# Harvest Site — Copilot Instructions

You are assisting in the **Harvest Website** — a Flutter web project that serves as the official landing page and Help Center for the Harvest e-commerce app.

Generate **high-quality, performant, and maintainable** code that follows the conventions established below.

---

## Project Overview

| Aspect | Detail |
|---|---|
| **Platform** | Web only |
| **SDK** | Dart ^3.11.4 |
| **Routing** | `go_router` (declarative, path-based) |
| **Fonts** | `google_fonts` — Poppins (headings), Inter (body) |
| **Theme** | Light-only Material 3, "Fresh Harvest" palette, custom `SiteColors` / `SiteTypography` / `SiteTheme` |
| **Linting** | `flutter_lints` |

---

## Brand

**Harvest** — Fresh from the farm. Direct to your community.

The site communicates Harvest's value proposition: connecting communities directly with local farmers for fresh, sustainable produce.

## Architecture

### Pattern — Page-based with shared widgets

```
lib/
├── main.dart                     # Entry point (MaterialApp.router)
├── router/
│   └── site_router.dart          # GoRouter config + route constants
├── theme/
│   ├── site_colors.dart          # Colour palette tokens
│   ├── site_theme.dart           # ThemeData builder
│   └── site_typography.dart      # Text style tokens
├── widgets/                      # Shared reusable widgets across pages
│   ├── site_header.dart          # Navigation header
│   ├── site_footer.dart          # Footer with links
│   └── ...                       # Other shared widgets
└── pages/
    ├── landing/
    │   ├── landing_page.dart     # Landing page (assembles sections)
    │   └── widgets/              # Landing-specific section widgets
    │       ├── hero_section.dart
    │       ├── features_section.dart
    │       ├── how_it_works_section.dart
    │       └── cta_section.dart
    └── help_center/
        ├── help_center_page.dart # Help center page (assembles sections)
        └── widgets/              # Help-center-specific widgets
            ├── help_hero.dart
            └── faq_section.dart
```

## Theme — "Fresh Harvest"

### Colour Palette (`SiteColors`)

| Token | Hex | Usage |
|---|---|---|
| `primary` | `#2D6A4F` | Primary CTAs, header highlights |
| `primaryLight` | `#52B788` | Badges, accents |
| `primaryDark` | `#1B4332` | CTA section background |
| `secondary` | `#E76F51` | Action buttons (Download, CTA) |
| `background` | `#FAFAF5` | Main background |
| `surface` | `#FFFFFF` | Cards, containers |
| `surfaceVariant` | `#F0EDE6` | Alternate sections |
| `onBackground` | `#1B1B1B` | Primary text |
| `onBackgroundLight` | `#6B7280` | Secondary text |

- **Light theme only** — consistent with the main Harvest app.
- **Never** use raw `Color(0x...)` in widgets — always use `SiteColors`.

### Typography (`SiteTypography`)

| Role | Font | Style |
|---|---|---|
| Display / Headlines | Poppins | Bold / SemiBold |
| Body text | Inter | Regular / Medium, line-height 1.5–1.6 |

- Text styles are accessed via `SiteTypography` — never inline `GoogleFonts.*()` calls in widgets.
- Use `Theme.of(context).textTheme.*` only from the `SiteTheme` configuration.

## Design Principles

- **DRY** — Extract shared widgets (header, footer) into `lib/widgets/`. Extract page sections into `pages/<page>/widgets/`.
- **KISS** — Simple site, no state management. Favour simplicity.
- **Composition over inheritance** — Prefer composing small widgets.
- **Responsiveness** — Every section must render well on mobile (< 600px) and desktop (> 900px).
- **Performance** — Use `const` constructors wherever possible.

---

## Coding Conventions

| Rule | Detail |
|---|---|
| **Naming** | `snake_case` files, `PascalCase` classes, `camelCase` members |
| **Imports** | Use package imports (`package:harvest_site/...`), not relative |
| **`const`** | Apply `const` constructors and `const` values wherever possible |
| **Private fields** | Prefix with `_`; expose only what's needed |
| **File length** | Keep files under 200 lines; split if larger |
| **Widget decomposition** | Extract widgets when they exceed ~50 lines or are reused |
| **Comments** | Only where the *why* isn't obvious; avoid restating the code |
| **Inline styles** | Never use `GoogleFonts.*()` directly in widgets — use `SiteTypography` |
| **Inline colours** | Never use `Color(0x...)` in widgets — use `SiteColors` |

---

## When Adding a New Page

1. Create a folder under `lib/pages/<page_name>/` with `<page_name>_page.dart`.
2. Create page-specific section widgets under `lib/pages/<page_name>/widgets/`.
3. Add the route in `lib/router/site_router.dart`.
4. Reuse shared widgets from `lib/widgets/` (header, footer, etc.).
5. Use `SiteTypography` for all text styles and `SiteColors` for all colours.
